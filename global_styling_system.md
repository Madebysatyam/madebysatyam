## Governance (read first)

This file is the **single source of truth** for all design tokens and styling rules in this project.

### Change policy

1. **Do not edit this file without explicit approval.** Any request to change `global_styling_system.md` must be reviewed first.
2. Before changing tokens here, the agent must:
   - Summarize what would change and why
   - List impacted files (`styles/*.css`, `tailwind.config.js`, components, pages)
   - Wait for your confirmation after thorough review
3. **After approval only:** update this file, then sync implementation files to match (never drift the opposite direction without discussion).
4. Page-specific layout belongs in `styles/homepage.css`, `styles/projects.css`, etc. — not in this document.

## Design principles

| Rule | Value |
|------|--------|
| Primary brand colour | **None** |
| UI base palette | **Space Grey only** |
| Default theme | **Dark** (`data-theme="dark"` or `:root`) |
| Light theme | Space Grey inverted (`data-theme="light"` or `prefers-color-scheme: light`) |
| Corner radius | **0 everywhere** (terminal sharp corners) |
| Accent palettes (blue, lime, pink, etc.) | **Content only** — projects, playground, tags; never nav, links, buttons, or site chrome |
| Markup | No inline styles; use tokens or Tailwind utilities |
| Type source | Derived from `Madebysatyam_Styles.json`, accessibility-tweaked |

## Build & usage

### Stylesheet entry

```html
<link rel="stylesheet" href="styles/global.css" />
```

`styles/global.css` imports: `color-tokens.css`, `layout-tokens.css`, `typography-tokens.css`, `text-styles.css`, `grid.css`.

### Tailwind (requires Node/npm)

```bash
npm install
npm run build:css   # compile
npm run watch:css   # develop
```

### HTML theme

```html
<html lang="en" data-theme="dark">
```

### Example markup

```html
<div class="container-site">
  <p class="text-style-label-large">Madebysatyam/</p>
  <h1 class="text-style-display-large">MADE BY SATYAM</h1>
  <p class="text-style-paragraph-medium">Body copy on space-grey-1000.</p>
</div>
```
